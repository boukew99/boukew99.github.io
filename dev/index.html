Overview & Zooming Interaction

When a user enters your site they should get an overview of the page. Minimize scrolling, to minimize overview time. Then the user can decide where to zoom for more information. Zooming can be done by loading a new page, as happens on the web when you click a hyperlink, or it could be a smooth zoom, such as on digital maps, which dynamically load in more details as you zoom in, to give you a new overview. Preferably you only want to zoom in, since zooming out means you did not find what you looked for.

Functional Foldering

Each page gets a folder to store its external files. The files will be owned by that page, but can still be accessed by other neighbour pages. Within that folder you can create another page to extend the page of the folder and recursively repeat the cycle again.

E-Mail

E-Mail is electronic mail and one of the older technologies. It is best used in exception cases, which don't require much scale. You can start a new inquiry or respond or forward a received e-mail. If you have more than one email, it will become a list of emails. You should see the subject first, then the senders email adress, then the message and closing with the name of the sender, if polite.

Sending a Email mirrors this with stating the subject first, if not responding or forwarding, and then inviting all the related email adresses. Then you write your messages and closing with your name. Add the current date if the timeframe of response is long, as to put an expirary date on the email.

On your site invite a email via form or with mailto:.
