Emotions in a logical environment

Computer software is an logical context-free language, there is only one way to interpret the meaning, independent of context. Though games have managed to evoke strong emotions in its players, which makes it stand out from other software applications. Those games are however made in the same environment as the rest of the applications, so how can they capture emotions so well? The trick is that the emotions are in-between the lines of code. This is why games are so mysteriously appealing and tricky to get right. It depends heavily on the involvement of the player to trigger the emotions, because the code cannot do it.

To make better videogames is to capture this emotion better. This requires cleaner minimal code, so the developers can iterate on the conditions of the gameplay that evoke the desired emotions.
